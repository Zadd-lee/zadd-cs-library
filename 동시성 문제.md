# 정의
여러 작업이 동시에 실행될 때 발생할 수 있는 예상치 못한 오류나 데이터 충돌 현상
공유 자원에서 여러 스레드, 혹은 사용자의 요청이 동시에 접근하면서 문제가 생김

# 대표적 증상
## Locs Update(업데이트 손실)
- 여러 개의 요청이 동시에 같은 데이터를 읽고 수정할 때 한 쪽의 수정 결과가 다른 쪽에 의해 덮어씌워져 사라지는 현상
## Race Condition
- 둘 이상의 작업이 동시에 실행되면서 실행 순서에 따라 결과가 달라지는 현상
- 공유 자원에 대해 동기화 없이 동시에 접근했을 때 작업 순서나 타이밍에 따라 결과 값이 달라짐
