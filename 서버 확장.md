# 서버를 확장
## 필요성
현대 웹 서비스 애플리케이션은 사용자의 요구가 늘어나고 비지니스 복잡도도 증가해 상당히 무거운 웹 애플리케이션으로 변화하고 있고 이에 따라 클라이언트의 수도 기하급수적으로 늘게 되어 기존보다 서버 부하 크게 증가함

## Scale up (수직 확장)
서버 자체의 사양을 높이는 것
기존 서버에 하드웨어 적으로 디스크를 추가하거나 cpu,ram 같은 자원을 업그레이드 하는 것

### 장점
추가적인 네트워크 연결 없이 서버의 자원을 늘릴 수 있음
수평적 확장(scale out) 방식보다 관리 비용이나 운영 이슈가 적고 확장 방식이 비교적 간단
서버의 개수가 1대 이므로 일관성이 유지됨
### 단점
하드웨어적 비용 부담이 크고 업그레이드 범위가 정해져 있어 성능 향상에 한계가 있음
서버 한 대가 모든 부하를 감당해야하고 서버가 터진다면 바로 서비스 장애로 이어짐
## Scale out (수평 확장)
서버의 개수를 늘리는 것
기존 서버와 비슷한 사양의 서버를 추가로 연결해 트래픽을 분산시켜 기존 서버의 부하를 줄이는 로드밸런싱을 사용
### 장점
유연한 확장성을 가짐
서버 한 대가 다운되어도 나머지 서버로 서비스를 운영할 수 있음
필요한 만큼 서버의 개수를 증설할 수 있어 확장에 제한 없음
### 단점
여러 노드를 연결해 병렬 컴퓨팅 환경을 구성해야 하기 때문에 관련 아키텍쳐에 대한 높은 이해 필요
데이터의 일관성이 깨질 수 있음
