# 정의
master에서 복제본으로 데이터가 동기화되는 과정에서 발생하는 시간 차이

## 원인
- 네트워크 지연(Network Latency)**: 마스터와 복제본 간 네트워크 대역폭 부족 또는 지연으로 명령 전송이 늦어짐.
- 복제본 처리 능력 부족(Replica Overload)**: 복제본이 읽기 트래픽(조회)을 많이 처리하거나, 복제 명령을 처리하는 데 부하가 걸릴 때 발생.
- 비동기 복제 (Asynchronous Replication)**: 레디스의 기본 복제 방식은 비동기라 마스터가 복제본의 커밋 여부와 상관없이 쓰기를 완료하므로 지연 발생 가능.
- 대량 데이터 전송/복구 시: 장애 후 재동기화(Full Resync) 시 대량의 데이터를 전송하면서 지연 발생.

복제 방식 변경
