# 다이나믹 프로그래밍
**다이나믹 프로그래밍 = 기억하며 풀기/ 기억하며 연산**
 
## 목적
수많은 경우의 수를 전부 확인해봐야 하는 상황에서 속도가 느려지는 문제를 개선하고자(수행시간을 단축하기 위해) 만들어진 알고리즘

메모리를 사용해 중복 연산을 줄이고 중복 연산을 줄여 수행 속도를 개선한다.
	메모리를 사용한다 = 배열 혹은 자료구조를 만든다
	중복 연산을 줄인다 = 연산한 결과를 배열에 담는다
	다시는 같은 연산을 하지 않는다

## 기준
1. DFS/ BFS로 풀 수는 있지만 경우의 수가 너무 많음
2. 경우의 수들에 중복적인 연산이 많은 경우

## 장점
이전에 연산한 값을 기반으로 새로운 값을 갱신할 수 있다
이전의 모든 조합 중 최고의 값만을 남겨놨기 때문에 이전의 정보를 볼 필요가 없어짐

## 문제해결 접근방법
_**최대한 많은 문제들을 풀며 풀이들을 참고하면서 DP식의 사고방식을 습득**_  - 오래 붙잡고 있기 보다는 30분 고민해보고 답이 안나오면 풀이를 참고해서 구현만 해보는 방법을 추천
"어떻게 하면 되로 돌아가지 않을까" - 이전의 연산을 또 하지 않으려면 어떤 정보들을 남겨야지? 어떤 식으로 정보를 누적해야 하지?

DP 삼각형과 같이 나만의 자료구조를 하나 더 만들고 어떤 정보를 담아야 이전 단계로 돌아가지 않을지 고민해보기
