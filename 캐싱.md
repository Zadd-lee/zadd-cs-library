자주 사용되는 데이터나 계산 결과의 복사본을 더 빠른 저장소(캐시) 에 임시로 저장하여, 이후 요청 시 원본 데이터에 접근하지 않고 빠르게 응답하는 기술

서버, 브라우저, 네트워크 장치, 디바이스등 여러 곳에 적용할 수 있다

서버에 적용하는 캐시는 서버사이드-캐싱 멀리 있는 서버의 데이터를 요청할 때, 캐싱으로 미리 근처 서버에 데이터를 뿌려놓고 가까이서 받을 수 있게함

속도가 장점이지만 보안과 업데이트 측면에서는 단점이다
캐시 데이터를 계속 저장하고 있으면 보안 문제가 생길 수도 있고, 데이터가 업데이트 돼서 무언가 바뀌었는데 캐시로 인해 바뀐 데이터가 아닌 기존 데이터를 보게되는 문제가 발생할 수 있음

이를 해결하기 위해 캐싱의 유효기간을 설정함
