많은 서버 용량과 성능을 필요할 때 인프라 확장 방식
# scale-up
기존 서버 사양을 업그레이드 해 시스템 확장하는 것
CPU, RAM 등 추가하거나 고성능의 부품, 서버로 교환하는 것

하나의 서버의 사양을 업그레이드 하기 때문에 수직 스케일로 불리기도 한다.

# scale-out
서버 여러대를 추가해 시스템을 확장하는 것
서버 여러대로 나뉘기 때문에 각 서버에 걸리는 부하를 균등하게 해주는 [[3. Resource/개발 문서/로드 밸런싱]]이 필수

여러 대의 서버로 나눠 시스템을 확장하기 때문에 수평 스케일로 불리기도 한다.
