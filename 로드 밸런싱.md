# 정의
네트워크 또는 서버에 가해지는 부하 트래픽을 분산시켜주는 기술

# 로드 밸런서

클라이언트와 서버 사이에 위치해 서버의 부하를 분산시키는 하드웨어나 소프트 웨어를 의미

## 종류
### L4 Load Balancer
[[3. Resource/개발 문서/OSI 7계층]] 에서 4계층인 [[3. Resource/개발 문서/OSI 7계층#전송 계층]]을 기반으로 해서 포트 번호를 바탕으로 부하를 분산
4계층 이하의 모든 정보를 갖고 있기 때문에 IP 주소와 포트 번호등을 데이터 활용

한 대의 서버가 각각 다른 포트 번호를 부여하여 다수의 서버 프로그램을 운영하고 있다면 L4 이상의 로드 밸런서를 사용해야 한다
### L7 Load Balancer
[[3. Resource/개발 문서/OSI 7계층]]에서 7계층인 [[3. Resource/개발 문서/OSI 7계층#응용 프로그램]] 계층을 기반으로 하기 때문에 HTTP 헤더와 [[3. Resource/개발 문서/쿠키]]와 같은 사용자의 요청 정보를 기준으로 특정 서버에 대한 부하를 분산
# 로드밸런싱 알고리즘
## 라운드 로빈
서버에 들어온 순서대로 돌아가며 배정하는 방식
클라이언트의 요청을 순서대로 분배하기 때문에 여러 대의 서버가 동일한 스펙을 갖고 있고, 서버와의 연결(세션)이 오래 지속되지 않는 경우에 활용
## 가중 라운드 로빈 방식
서버마다 가중치를 매기고 가중치가 높은 서버에 요청을 우선적 배분
서버의 트래픽 처리 능력이 상이한 경우 사용

## ip 해시 방식
클라이언ㅌ의 ip주소를 특정 서버로 매핑해 처리
사용자의 ip를 해싱해 분배하기 때문에 사용자가 항상 동일한 서버로 연결되는 것을 보장
### 해싱
임의의 길이를 지닌 데이터를 고정된 길이의 데이터로 매핑하거나 그러한 함수

## 최소 연결 방식
요청이 들어온 시점에서 가장 적은 연결상태를 보이는 서버에 우선적으로 트래픽을 배분
자주 세션이 길어지거나, 서버에 분배된 트래픽들이 일정하지 않은 경우 적합
## 최소 리스폰타임
서버의 현재 연결 상태와 응답 시간을 모두 고려해 트래픽을 배분
가장 적은 연결 상태와 가장 짧은 응답시간을 보이는 서버에 우선적으로 로드를 배분
### 응답시간
서버에 요청을 보내고 최소 응답을 받을 때까지 소요되는 시간
