# 우선순위 큐
Queue : 먼저 들어오는 데이터가 먼저 나가는 FIFO(First In First Out)
우선순위 큐 : 먼저 들어오는 데이터가 아닌 우선순위가 높은 데이터가 먼저 나가는 구조 
일반적으로 힙을 이용해 구현한다

# 힙
우선순위 큐를 위해 고안된 완전이진트리 형태의 자료구조
여러 개의 값 중 최댓값 또는 최솟값을 찾아내는 연산이 빠르다
## 특징
[[이진트리#완전이진트리]] 형태로 이루어져 있음
부모노드와 서브트리간 대소관계가 성립(반정렬상태)
이진탐색트리(BST)와 달리 중복값 허용됨

## 종류
### 최대 힙
부모 노드의 키 값이 자식 노드보다 크거나 같은 완전 이진트리
>[!info] key(부모노드) $\leq$ key(자식 노드)
### 최소힙
부모 노드의 키 값이 자식 노드보다 작거나 같은 완전 이진트리
>[!info] key(부모노드) $\geq$ key(자식 노드)

### 우선 순위 큐 구현 방법 비교
힙이 아닌 배열 또는 연결 리스트를 이용해 구현 가능

| 구현 방법        | 삽입          | 삭제          |
| ------------ | ----------- | ----------- |
| 순서 없는 배열     | $O(1)$      | $O(n)$      |
| 순서 없는 연결 리스트 | $O(1)$      | $O(n)$      |
| 정렬된 배열       | $O(n)$      | $O(1)$      |
| 정렬된 연결 리스트   | $O(n)$      | $O(1)$      |
| 힙            | $O(log_2N)$ | $O(log_2N)$ |
## 구현
- 우선큐에 저장할 객체는 필수적으로 Comparable Interface를 구현해야함
	- Comparable interface의 compareTo method를 오버라이딩 하게 되고 처리할 우선 순위 조건을 리턴해주면 PriorityQueue 가 알아서 우선순위가 높은 객체를 추출함
- Heap 을 통해 구현하는 것이 일반적임
```java
import java.util.PriorityQueue;
import java.util.Collections;

//낮은 숫자가 우선 순위인 Integer 형 우선순위 큐 선언
```java
PriorityQueue<Integer> priorityQueueLowest = new PriorityQueue<>();

//높은 숫자가 우선 순위인 int 형 우선순위 큐 선언
PriorityQueue<Integer> priorityQueueHighest = new PriorityQueue<>(Collections.reverseOrder());

//값 추가
priorityQueueLowest.add(1);  //문제 상황시 예외 리턴
priorityQueueLowest.offer(3); // 문제 상황시 값 리턴

//값 삭제
priorityQueueLowest.remove(); //문제 상황시 예외 리턴
priorityQueueLowest.poll(); //문제 상황시 값 리턴

//값 검사
priorityQueueLowest.element(); //문제 상황시 예외 리턴
priorityQueueLowest.peek(); //문제 상황시 값 리턴

// 초기화
priorityQueueLowest.clear();

```
