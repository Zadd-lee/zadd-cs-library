# HTTP(HyperText Tranfer Protocol)
서버와 클라이언트가 서로 데이터를 주고 받기 위해 사용되는 통신 규약
요청과 응답 구조를 가짐
## 특징
- 클라이언트 - 서버 모델 : 사용자의 웹 브라우저가 서버에 요청을 보내면, 서버가 응답을 보내는 방식
- 요청 - 응답 메세지 : 클라이언트의 get,post 등의 요청과 서버의 200 ok 같은 상태 코드 및 데이터를 담은 응답 메세지로 구성
- 다양한 데이터 전송 : html 뿐만 아니라 이미지, 동영상, json등 거의 모든 데이터 전송 가능
- 상태 비저장(stateless) : 이전 통신 내용을 기억하지 못함 이를 보안하기 위해 쿠키, 세션 등 기술이 사용됨
- 포트 : 일반적으로 80번 포트 사용
